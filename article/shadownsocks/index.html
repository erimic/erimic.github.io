Debian下Shadowsocks安装及优化

本文主要针对Xen架构vps环境搭建Shadowsockes服务器。并启用Google BBR优化内核中的拥塞算法，优化TCP。

一键安装脚本参照的是秋水逸冰 的博文，出于低内存低CPU考虑，使用shadowsocks-libev

Shadowsocks安装

安装方法

使用root用户登录

wget --no-check-certificate https://raw.githubusercontent.com/teddysun/shadowsocks_install/master/shadowsocks-libev-debian.sh
chmod +x shadowsocks-libev-debian.sh
./shadowsocks-libev-debian.sh 2>&1 | tee shadowsocks-libev-debian.log

配置文件路径

/etc/shadowsocks-libev/config.json

内容如下

{
    "server":["[::0]","0.0.0.0"],
    "server_port":8989,
    "local_address":"127.0.0.1",
    "local_port":1080,
    "password":"password",
    "timeout":600,
    "method":"aes-256-cfb"
}

各字段含义

server：服务器 IP (IPv4/IPv6)，注意这也将是服务端监听的 IP 地址

server_port：监听的服务器端口

local_address：本地监听的 IP 地址

local_port：本地端端口

password：用来加密的密码

timeout：超时时间（秒）

method：加密方法，可选择 “bf-cfb”, “aes-256-cfb”, “des-cfb”, “rc4”, 等等。默认是一种不安全的加密，推荐用 “aes-256-cfb”

卸载方法

./shadowsocks-libev-debian.sh uninstall

常用命令

启动:

/etc/init.d/shadowsocks start

停止:

/etc/init.d/shadowsocks stop

重启:

/etc/init.d/shadowsocks restart

查看状态:

/etc/init.d/shadowsocks status

Shadowsocks客户端配置

Windows下客户端较多，我们采用GitHub上开源的shadowsocks-windows下载地址:

https://github.com/shadowsocks/shadowsocks-windows/releases

解压至任意目录下，双击Shadowsocks.exe运行程序，填入各参数

一键安装最新内核并开启BBR

参考秋水逸冰 的另一博文一键安装最新内核并开启BBR

使用root用户登录

wget --no-check-certificate https://github.com/teddysun/across/raw/master/bbr.sh
chmod +x bbr.sh
./bbr.sh

安装后重启VPS验证是否启用BBR

uname -r

4.10.1-041001-generic

sysctl net.ipv4.tcp_available_congestion_control

net.ipv4.tcp_available_congestion_control = bbr cubic reno

sysctl net.ipv4.tcp_congestion_control

net.ipv4.tcp_congestion_control = bbr

sysctl net.core.default_qdisc

net.core.default_qdisc = fq

lsmod | grep bbr

返回有值表明已启用BBR算法

系统优化

increase the maximum number of open file descriptors

编辑limits.conf

vi /etc/security/limits.conf

添加一下两行

* soft nofile 51200
* hard nofile 51200

ulimit -n 51200

重启shadowsocks

Tune the kernel parameters

编辑sysctl.conf

vi /etc/sysctl.conf

fs.file-max = 51200

net.core.rmem_max = 67108864
net.core.wmem_max = 67108864
net.core.netdev_max_backlog = 250000
net.core.somaxconn = 4096

net.ipv4.tcp_syncookies = 1
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_tw_recycle = 0
net.ipv4.tcp_fin_timeout = 30
net.ipv4.tcp_keepalive_time = 1200
net.ipv4.ip_local_port_range = 10000 65000
net.ipv4.tcp_max_syn_backlog = 8192
net.ipv4.tcp_max_tw_buckets = 5000
net.ipv4.tcp_fastopen = 3
net.ipv4.tcp_mem = 25600 51200 102400
net.ipv4.tcp_rmem = 4096 87380 67108864
net.ipv4.tcp_wmem = 4096 65536 67108864
net.ipv4.tcp_mtu_probing = 1
